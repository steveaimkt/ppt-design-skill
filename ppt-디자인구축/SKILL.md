# getdesign.md로 짓는 PPT 디자인 시스템 (ppt-디자인구축, 방법①)

> **게이트형. 자동 실행 금지.** 각 PHASE 끝에서 사용자에게 묻고, 답을 받은 뒤 다음으로.
> (상시 지시: 디자인 방향·범위·결정은 무조건 먼저 물어본다.)

## 핵심 메시지
> **디자인을 새로 그리지 않는다.** getdesign.md에서 좋은 디자인 시스템을 **고르고** →
> 우리 회사 옷을 **입혀** → 엑셀로 PPT를 만든 뒤 → 그 흐름을 **스킬로 굳히면**, 다음 PPT는 엑셀만 갈아 끼우고 한 줄.

## 경로 (이 SKILL.md 기준)
- 우리 톤 지침 완성 예시: `../../../자료/PPT방법1_클로드프로젝트_지침_N식품.md`
- 샘플데이터: `sample-data/` (2교시와 동일한 엑셀 6종 — 스킬에 번들)
- 치트시트: `../../../자료/프롬프트_치트시트_문서.md`

## PHASE ① 디자인 고르기 — getdesign.md 직접 방문 (STEP 1)
> 📄 **getdesign.md** = 전 세계 수십 개 브랜드의 디자인 시스템을 프롬프트로 정리해 둔 사이트.

사용자가 **직접 사이트에 들어가 고르고 복사해 붙여넣는** 과정으로 진행한다.

1. **사이트 직접 접속**: 브라우저에서 https://getdesign.md/ 를 연다.
2. **고르기 → 복사 (3클릭)**: ① 마음에 드는 브랜드/디자인 클릭 → ② `design.md` 버튼 열기 → ③ **Copy**로 프롬프트 전체 복사.
   - 톤 가이드: 미니멀·절제 / 금융 신뢰감 / 묵직한 임원 보고 등 — 원하는 인상으로 고른다.
3. **여기 채팅창에 붙여넣기**: 복사한 디자인 프롬프트(영문 구조)를 그대로 붙여넣어 전달한다.

→ 붙여넣은 프롬프트를 확인하면 PHASE ②(우리 옷 입히기)로.

## PHASE ② 우리 것으로 — 색·폰트·톤 (STEP 2)
복사한 프롬프트에 우리 회사 옷을 입힌다. 핵심은 **한 줄 지시 + 컴플라이언스 내장**.
```
이 디자인 프롬프트를 N식품 스타일로 바꿔줘 —
색은 N Green(#00843D)+Point Orange(#F47920), 폰트는 Pretendard(한글),
톤은 정중·묵직한 임원 보고용, 그리고 식품 컴플라이언스 5원칙을 자동 내장.
```
- 산출물 = 우리 회사 **디자인 지침**(완성 형태는 `자료/PPT방법1_클로드프로젝트_지침_N식품.md` 참고:
  ROLE·OVERVIEW·COLORS·TYPOGRAPHY·LAYOUT·표준 8섹션·톤·컴플라이언스 가드).
- 원본의 영어 구조는 두고 "우리 스타일로"만 지시하면 나머지는 클로드가 채운다.

→ 지침 초안 확인 후 PHASE ③.

## PHASE ③ 한 장 테스트 — 피드백 루프 (STEP 3)
한 번에 완성하지 않는다. **한 장 만들어 보고, 보면서 고친다.**
1. 표지 또는 데이터 슬라이드 **한 장**만 생성.
2. 결과를 **화면 캡처**해 클로드에 그대로 붙인다.
3. "제목 더 크게 · 색 한 톤 낮춰"처럼 **콕 집어** 요청 → 반영 → 반복(보통 2~3회).

→ 한 장이 만족스러우면 PHASE ④.

## PHASE ④ 본격 사용 — 2교시 엑셀 샘플 → PPT (STEP 4)
> **2교시와 같은 엑셀로 이번엔 PPT를 만든다.** 2교시는 같은 데이터로 한글 보고서를 만들었다 —
> 3교시는 같은 데이터에 우리 디자인 시스템을 입혀 **제안서·보고 PPT**로. 데이터는 그대로, 포맷만 바뀐다.

1. **데이터 선택 (게이트)**: 아래 `sample-data/` 6종을 표로 보여주고, **어떤 데이터로 PPT를 만들지 사용자에게 묻는다(AskUserQuestion).** 답을 받은 뒤 다음 단계로(없으면 본인 업무 엑셀).

| 파일 | 핵심 컬럼 | 내장 스토리 | 권장 PPT |
|---|---|---|---|
| 영업_거래처별매출.xlsx | 거래처·매출·목표·반품 | 가온마트 급성장 | **입점·매출 제안서** ★ |
| MD_상품별판매.xlsx | SKU·판매·재고·반품 | 단호박스프 급상승 | **신상품 기획서·SKU 분석** ★ |
| 마케팅_캠페인성과.xlsx | 캠페인·채널·광고비·매출 | ROAS 0.6~7.4 편차 | **캠페인 성과 보고서** ★ |
| 물류_입출고재고.xlsx | 입출고·재고·리드타임 | 품절위험 4종 | 재고 운영 (HWP 권장) |
| 품질_점검로그.xlsx | 점검·결과·심각도 | 부적합 9·미조치 3 | 점검 결과 (HWP 권장) |
| 생산_라인별일지.xlsx | 계획·생산·불량·가동 | 3라인 저조 | 생산 실적 (HWP 권장) |

2. **데이터 확인**: 고른 엑셀을 읽고 핵심 숫자 5개·차트거리 정리(근거 먼저).
3. **한 줄 요청 → PPT 생성**(글로벌 `pptx` 스킬):
   ```
   위 디자인 시스템을 적용해, 이 엑셀(예: 영업_거래처별매출.xlsx)로 [문서종류] PPT(.pptx)를 만들어줘.
   표지·핵심요약·현황·분석·제안·기대효과 흐름, 한 슬라이드 한 메시지, 핵심 숫자는 차트로.
   ```
4. **검산·보강**: 표·차트 수치를 원본 엑셀과 대조. "AI 티" 제거(문단 통짜·점점점 나열·과한 그림자 금지).
5. 첫 PPT가 나왔습니다. 이제 이 흐름을 재사용 자산으로 굳힐 차례.

→ PPT 확인 후 PHASE ⑤.

## PHASE ⑤ 스킬로 등록 (STEP 5, 마지막)
지금까지 만든 **디자인 시스템 + 엑셀→PPT 흐름**을 재사용 스킬로 굳힌다.

1. 사용자에게 **반드시 묻는다**:
   > "지금 진행한 내용(디자인 시스템 + 엑셀→PPT 제작 흐름)을 **스킬로 만들겠습니까?**"
2. 동의하면 새 스킬을 구축한다 — 예: `skills/<부서>-ppt/SKILL.md`
   - PHASE ②~③에서 만든 **우리 디자인 지침을 본문에 내장**(또는 `자료/` 경로로 링크)
   - 트리거(예: "<부서> PPT 만들자"), 게이트 PHASE(엑셀 선택 → 분석 → 디자인 적용 PPT 생성 → 검산)
   - 글로벌 `pptx` 의존성 명시
3. 이후부터는 그 스킬 **한 번 호출 → 엑셀만 갈아 끼우면** 일관 톤 PPT.
→ 스킬 생성 확인 후 종료.

## 표준 구조 8섹션 (지침에 내장)
표지 → 핵심 요약 → 배경·현황 → 분석·시사점 → 제안·방안 → 기대효과 → 추진 일정 → 부록·출처.
(한 줄 결론 → 근거 3 → "그래서 X" / 데이터 카드·차트 / 옵션 카드 / 핵심 수치 풀스크린)

## 산출물 규칙
- 완성도 기준 = `자료/PPT방법1_클로드프로젝트_지침_N식품.md` 디자인 시스템 100% 준수.
- 표 숫자는 원본 엑셀과 일치(원본 대조). 익명화(N식품/WMBB 가상) 준수.
- **식품 컴플라이언스 가드**: 과대광고 표현 회피("최고"·"100% 안전"·효능 단정 금지),
  거래처·개인정보 마스킹("○○마트"→"A 거래처"), 확정 수치는 사람이 최종 검수.

## 방법① vs 방법② (오해 방지)
- **방법①(본 스킬)**: 디자인 시스템을 **스킬로 등록** → 빠르고 비개발자 친화. 톤만 맞으면 충분할 때.
- **방법②(4교시 `ppt-만들기`)**: 디자인을 **직접 만들어 코드(토큰·템플릿)로 고정** → 일관성·세밀 제어·대량 생산.
